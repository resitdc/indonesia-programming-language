Rakoda Programming Language (RPL)

Rakoda Programming Language (RPL) adalah bahasa pemrograman yang dirancang khusus untuk mempermudah siswa SMK (khususnya jurusan RPL) dalam memahami esensi algoritma pemrograman tanpa kerumitan sintaksis asing. Menggunakan kata-kata instruksi berbahasa Indonesia yang natural, RPL memadukan kemudahan belajar dengan performa tangguh berbasis Bytecode Virtual Machine (VM).

Sintaks Bahasa Indonesia

Logika alur menggunakan kata seperti jika, fungsi, dan selesai agar intuitif bagi pemula.

Mesin Virtual Kustom

Eksekusi super cepat dengan parser, compiler bytecode, dan VM kustom berbasis Rust dengan Garbage Collector (GC).

Siap untuk Web Modern

Dilengkapi web server native berkecepatan tinggi dengan fitur kompresi, rate limiter, proxy, cookie, dan session.

💡 Mengapa RPL?

Siswa seringkali kesulitan mempelajari konsep lanjutan seperti Web Routing, API Gateway, atau Session karena sintaksis asinkronus (async/await) yang berbelit. RPL menyembunyikan kompleksitas tersebut secara aman lewat pendekatan konkuren yang sinkronus.

Instalasi & CLI

Ekosistem RPL dibangun menggunakan Rust dan didistribusikan sebagai satu file executable rpl yang serbaguna.

Menjalankan Kode Program

Gunakan perintah run untuk mengeksekusi file program berekstensi .rpl:

rpl run program.rpl

Watch Mode (Hot Reload)

Ingin server otomatis restart saat Anda mengedit kode? Tambahkan argumen --watch atau -w:

rpl run program.rpl --watch

Sub-perintah CLI

Perintah CLI Deskripsi
rpl run <file> Mengeksekusi kode RPL menggunakan Virtual Machine (Default).
rpl run <file> --interpreter Mengeksekusi kode langsung lewat pohon AST (Alternatif).
rpl repl Membuka mode interaktif Command Line (REPL).
rpl fmt <file> Merapikan format penulisan file RPL otomatis.

Variabel & Tipe Data

Di dalam RPL, variabel dideklarasikan menggunakan kata kunci buat dan bersifat dinamis (*dynamically typed*).

// Contoh Deklarasi Variabel
buat nama = "Restu Dwi"
buat umur = 26
buat adalahAdmin = benar
buat dataSiswa = kosong // Menyatakan ketiadaan nilai (null)

Tipe Data Utama

  • Angka (Number): Bilangan desimal maupun bulat (contoh: 10, 3.14).
  • Teks (String): Kumpulan karakter dibungkus tanda kutip ganda (contoh: "Halo Dunia").
  • Boolean: Nilai logika, ditulis dengan benar (true) atau salah (false).
  • Daftar (Array): Kumpulan elemen berindeks dimulai dari 0 (contoh: [1, "dua", benar]).
  • Kamus (Kamus/Dictionary): Pasangan kunci dan nilai (contoh: { "nama": "Ahmad", "nilai": 90 }).

Mengakses Kamus & Array

buat nilaiSiswa = [80, 95, 70]
tampilkan nilaiSiswa[1] // Output: 95

buat profil = { "kota": "Bandung", "sekolah": "SMK 1" }
tampilkan profil.kota     // Output: Bandung
tampilkan profil["kota"]  // Output: Bandung

Operator

Gunakan operator aritmatika, perbandingan, dan logika bawaan untuk mengolah data.

Operator Aritmatika

Mendukung tambah (+), kurang (-), kali (*), bagi (/), dan sisa bagi (% atau keyword mod).

Operator Perbandingan & Logika

Operator Artinya Keyword Alternatif
== Sama Dengan -
!= Tidak Sama Dengan -
>, >= Lebih Dari, Minimal minimal
<, <= Kurang Dari, Maksimal maksimal
dan Logika DAN (AND) -
atau Logika ATAU (OR) -
bukan Negasi / Kebalikan (NOT) -

Kondisi & Perulangan

RPL mendukung penulisan blok kode kontrol alur yang fleksibel. Anda dapat memilih menggunakan tanda kurung kurawal {} atau kata kunci maka yang diakhiri dengan selesai.

Kondisi (Percabangan)

Gaya Kata Kunci (maka...selesai)
jika umur >= 17 maka
    tampilkan "Bisa membuat KTP"
jika tidak
    tampilkan "Belum cukup umur"
selesai
Gaya Kurawal {}
jika umur >= 17 {
    tampilkan "Bisa membuat KTP"
} jika tidak {
    tampilkan "Belum cukup umur"
}

Perulangan (Loop)

Perulangan menggunakan instruksi selama:

buat hitung = 1
selama hitung <= 3 maka
    tampilkan "Hitungan ke: " + hitung
    hitung = hitung + 1
selesai

Penanganan Error (Exception)

Bahasa pemrograman RPL mendeteksi error secara cerdas dan memberikan pesan ramah pemula. Anda juga dapat menggunakan blok coba-tangkap untuk mencegah aplikasi terhenti saat menemui kegagalan.

Blok Coba-Tangkap (Try-Catch)

fungsi pembagian(a, b)
    jika b == 0 maka
        lempar "Kesalahan: Pembagian dengan nol!"
    selesai
    kembalikan a / b
selesai

coba
    buat hasil = pembagian(10, 0)
    tampilkan hasil
tangkap (error)
    tampilkan "Error ditangkap: " + error
selesai

Konkurensi (Tugas Background)

Ingin menjalankan beberapa operasi berat bersamaan (seperti fetch API atau query file) tanpa menangguhkan web server utama? RPL menyediakan modul tugas.

// Jalankan tugas secara paralel di thread terpisah
buat tugas1 = tugas.jalankan(fungsi() {
    waktu.tunggu(2000) // Tidur 2 detik
    kembalikan "Tugas 1 Selesai!"
})

buat tugas2 = tugas.jalankan(fungsi() {
    waktu.tunggu(1000) // Tidur 1 detik
    kembalikan "Tugas 2 Selesai!"
})

// Tunggu hasilnya saat dibutuhkan
tampilkan tugas.tunggu(tugas2)
tampilkan tugas.tunggu(tugas1)

Modul Bawaan

RPL menyediakan sekumpulan modul siap pakai untuk menyelesaikan tugas-tugas pemrograman sehari-hari.

matematika

Melakukan pengolahan angka.

Fungsi Hasil / Contoh Penggunaan
matematika.tambah(a, b) Mengembalikan hasil penjumlahan a + b.
matematika.kurang(a, b) Mengembalikan hasil pengurangan a - b.
matematika.kali(a, b) Mengembalikan hasil perkalian a * b.
matematika.bagi(a, b) Mengembalikan hasil pembagian a / b.
matematika.pangkat(basis, eksponen) Mengembalikan nilai pangkat basis^eksponen.
matematika.bulatkan(x) Membulatkan angka desimal terdekat (contoh: 3.5 -> 4).
matematika.acak(min, max) Menghasilkan angka acak antara min dan max.
matematika.akar(x) Menghitung akar kuadrat dari x.

string

Memanipulasi dan mengonversi tipe teks.

Fungsi Deskripsi / Hasil
string.panjang(teks) Mengembalikan panjang karakter dari teks.
string.besar(teks) Mengubah teks menjadi huruf KAPITAL semua.
string.kecil(teks) Mengubah teks menjadi huruf kecil semua.
string.potong(teks, mulai, selesai) Memotong substring dari indeks mulai hingga selesai.
string.ganti(teks, cari, ganti) Mengganti bagian teks tertentu dengan teks baru.
string.dari(nilai) Mengonversi tipe data apa pun (Kamus, Array, dll) menjadi teks String.

list

Mengelola struktur data Array/Daftar.

  • list.tambah(array, elemen): Menyisipkan elemen baru ke bagian akhir array.
  • list.hapus(array, indeks): Menghapus elemen pada indeks tertentu dan mengembalikannya.
  • list.panjang(array): Mengambil total jumlah elemen di dalam array.
  • list.ambil(array, indeks): Mengambil elemen pada indeks tanpa menghapusnya.

json

Melakukan konversi struktur data Kamus/Array RPL ke format teks JSON.

buat user = { "id": 1, "nama": "Rian" }

// Serialisasi ke JSON string
buat json_str = json.stringify(user) 
tampilkan json_str // Output: {"id":1,"nama":"Rian"}

// Deserialisasi dari JSON string kembali ke Kamus RPL
buat data_kamus = json.parse(json_str)
tampilkan data_kamus.nama // Output: Rian

waktu

Mengambil parameter waktu sistem.

  • waktu.sekarang(): Timestamp Unix saat ini (detik).
  • waktu.string(): Format tanggal & jam terbaca (YYYY-MM-DD HH:MM:SS).
  • waktu.tunggu(milidetik): Menunda eksekusi proses selama durasi yang ditentukan.
  • waktu.tahun(), waktu.bulan(), waktu.tanggal(): Mendapatkan kalender hari ini.
  • waktu.jam(), waktu.menit(), waktu.detik(): Mendapatkan waktu jam saat ini.

file

Membaca dan menulis berkas lokal.

// Menulis file
file.tulis("catatan.txt", "Belajar RPL menyenangkan")

// Memeriksa file
jika file.ada("catatan.txt") maka
    // Membaca file
    buat isi = file.baca("catatan.txt")
    tampilkan isi
selesai

// Menghapus file
file.hapus("catatan.txt")

env

Mengelola konfigurasi Environment Variables.

  • env.get("NAMA_KUNCI"): Mengambil nilai *env* sistem.
  • env.set("NAMA_KUNCI", "Nilai"): Menetapkan/Mengubah nilai *env*.
  • env.load(".env"): Membaca berkas `.env` eksternal dan memuat isinya ke variabel lingkungan secara massal.

http (Client)

Mengirimkan permintaan jaringan (*HTTP requests*) ke web service eksternal.

buat respon = http.get("https://api.github.com/users/resitdc")
tampilkan "Status API: " + respon.status
tampilkan "Hasil Body: " + respon.body

RPL Web Server

RPL menyediakan server web native berperforma tinggi. Anda dapat mendefinisikan rute dan langsung meluncurkan web server hanya dengan beberapa baris kode.

// Mendefinisikan Rute GET
web.get("/", fungsi(req) {
    kembalikan "
Halo dari RPL Web Server!
"
})

// Menerima rute dengan data JSON / Tubuh request
web.post("/api/data", fungsi(req) {
    buat data = json.parse(req.tubuh)
    tampilkan "Menerima kiriman: " + data.nama
    kembalikan { "status": "sukses", "pesan": "data disimpan" }
})

// Jalankan server di port 4000
web.jalankan(4000)
🚀 Hot Reload Otomatis

Jalankan server Anda dengan perintah rpl run server.rpl --watch. Setiap kali Anda mengubah kode rute, server akan merestart otomatis secara instan dalam milidetik!

Fitur Lanjutan Web Server

Untuk kebutuhan produksi, RPL dibekali fitur keamanan dan optimasi tangguh di balik layar yang dapat dikonfigurasi langsung dari file kode RPL Anda.

1. Kompresi Otomatis (Gzip / Brotli)

Mengompresi dokumen HTML/JSON secara on-the-fly jika didukung oleh browser klien (berdasarkan header Accept-Encoding):

web.kompresi(benar) // Mengaktifkan optimasi bandwidth

2. Rate Limiting (Pencegah DDoS)

Membatasi request berlebihan dari satu IP address per detik untuk mengamankan server dari banjir traffic:

web.ratelimit(5) // Maksimal 5 permintaan/detik per IP. Melampaui ini mendapat status 429.

3. Reverse Proxy (API Gateway)

Meneruskan path request tertentu ke backend eksternal. Berguna untuk menyembunyikan API utama atau menghindari isu CORS:

// Meneruskan request yang berawalan /api ke server JSONPlaceholder secara otomatis
web.proxy("/api", "https://jsonplaceholder.typicode.com")

Cookie & Session Management

Kini Anda dapat membuat aplikasi web interaktif yang memiliki fitur login (*stateful*) menggunakan manajemen cookie dan session yang aman.

Penggunaan Cookie

// Menyimpan cookie di browser (max_age 3600 detik/1 jam bersifat opsional)
cookie.set("tema", "gelap", 3600)

// Mengambil cookie yang dikirim dari browser
buat tema_pengguna = cookie.get("tema")

// Menghapus cookie dari browser
cookie.hapus("tema")

Penggunaan Session (Auto-Managed)

Session di RPL sangat praktis karena browser dan server terhubung otomatis melalui cookie RPL_SESSIONID tanpa konfigurasi database tambahan:

// 1. Menyimpan data login ke session di server
session.set("user_id", 1002)
session.set("peran", "admin")

// Mengatur umur session (misal aktif selama 1 jam)
session.set_expired(3600)

// 2. Mengambil data session di rute lain
buat user = session.get("user_id")
jika user == kosong maka
    tampilkan "Silakan login terlebih dahulu!"
selesai

// 3. Menghapus data session saat logout
session.hapus("user_id")
